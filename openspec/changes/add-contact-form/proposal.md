# Change: Add Contact Form

## Why
The application needs a user-facing contact form to enable visitors to submit inquiries, feedback, or support requests. This is a foundational capability for user engagement and communication.

## What Changes
- Create a new contact form UI component with fields for name, email, subject, and message
- Implement client-side form validation with clear error messaging
- Add form submission handling with success/error feedback
- Implement accessibility features (ARIA labels, keyboard navigation, screen reader support)
- Add responsive design for mobile and desktop views
- Include basic input sanitization and validation rules

## Impact
- Affected specs: `contact-form` (new capability)
- Affected code: New frontend component, validation utilities, and submission handler
